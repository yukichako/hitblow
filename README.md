# hitblow（チームで育てる Hit & Blow）

第2回のチーム開発のスタート地点。**最初から遊べる**：

```bash
uv sync                       # pytest など開発環境を用意
uv run python -m hitblow      # 遊ぶ
uv run pytest                 # core のテスト（緑）
```

## 構成（責務分離）

```
src/hitblow/
├─ core.py   判定と出題（純粋関数・テスト対象）
├─ game.py   進行ループ ★機能はここの「ここに足す」場所に書く
├─ cli.py    main()（第3回で hitblow コマンドが呼ぶ）
└─ __main__.py
tests/test_core.py
```

## 並行作業をスムーズにするルール（重要）

2ペア（4名なら2+2、5名なら2+3）が**別々の機能**を同時に作ります。衝突で詰まらないために：

1. **1機能 = 1ファイル**：自分の機能は**新しいファイル**に書く（例 `hint.py` / `score.py`）。
   → 機能のコードは別ファイルなので**ぶつからない**。
2. **`game.py` は自分の担当の場所だけ直す**：`game.py` には「ここに足す」場所が3か所
   （① 開始時／② 入力コマンド／③ 勝利時）。担当の場所だけに書く。
   **import も自分の場所の近くに**書く（先頭にまとめない）。
3. **このリストにだけ1行足す**（↓）。ここは2ペアが触るので**コンフリクトが起きる前提**。
   起きたら「両方残す」で直す（＝コンフリクト練習）。

## 進め方
- 仕様を書く → **AI に実装させる**（コースサイトの「AI の使い方」参照）→ 遊んで確認＋`uv run pytest`。
- push → **PR** → 相手ペアが「動く・読める・**説明できる**」をレビュー → merge → 全員 `pull`。
- 1本目を merge したら、2本目のペアは `git pull` してから直す（上のリストがコンフリクトする）。
